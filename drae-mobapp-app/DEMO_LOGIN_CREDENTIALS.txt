DRAE MobApp — demo Supabase Auth logins
========================================

IMPORTANT---------
Running schema.sql / dummy_data.sql only fills tables like public.profiles.
It does NOT create accounts in Authentication. If you skip the steps below,
sign-in will always say “Invalid login credentials”.

Option A — Admin web (easiest)
------------------------------
1) Start the admin site (see START_GUIDE.txt) with the same Supabase .env as the mobile app.
2) Sidebar → “App logins”.
3) Pick a resident (e.g. Ana Santos), set login email (e.g. resident@drae.demo).
4) Click “Create login & link profile”. Copy the temporary password shown (it is not saved).
5) Resident opens the app, signs in with that email + temporary password, then sets a new password on the next screen.
6) In Supabase: Authentication → Providers → Email → disable “Confirm email” for local testing
   (or the new user may not be able to sign in until the email is confirmed).

Option C — Self sign-up in the mobile app
-----------------------------------------
1) Open the app → “CREATE ACCOUNT”.
2) Use the same email as your row in public.profiles (e.g. juan.delacruz@email.com from seed data).
3) Choose a password (min. 6 characters). If email confirmation is off, the app links your Auth user to
   that profile and continues. If confirmation is on, confirm from email, then use Log in — linking runs
   on first successful sign-in.
4) Supabase Auth must accept the address; some fictional domains fail — staff can update profiles.email
   to a real domain or you test with Dashboard / App logins instead.

Option B — Supabase Dashboard
-----------------------------
1) Dashboard → Authentication → Users → “Add user”.
2) Email: resident@drae.demo
   Password: DraeResident2026!
   Turn ON “Auto Confirm User” (or disable email confirmation — see below).
3) Repeat for staff@drae.demo / DraeStaff2026!

4) Dashboard → SQL → run supabase/link_demo_auth_users.sql
   (links each auth user to the seeded profile row). Skip this if you used Option A (linking is automatic).

5) If you still see “Email not confirmed”:
   Authentication → Providers → Email → disable “Confirm email” for dev.

Notes
-----
- Login email can differ from the profile row’s display email in seed data.
- Accounts created in the Dashboard (Option B) use the password you type and are not flagged for a forced change. Admin “App logins” (Option A) always uses a generated temporary password and sets must_change_password on the profile.
- Use “Add user” with a password, not only “Invite”, unless you complete the invite flow.
- Ensure the app’s .env EXPO_PUBLIC_SUPABASE_URL / ANON_KEY are from this same project.
