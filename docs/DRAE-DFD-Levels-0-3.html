DRAE Platform — Data Flow Diagrams

Context through Level 3 for the City of Dasmariñas disaster risk awareness ecosystem (resident mobile app, CDRRMO admin web, Supabase backend). Notation: rounded = process, square = external entity, open rectangle = data store, arrows = data flows.

Jump to Level 0 — Context Level 1 Level 2 Level 3

Level 0 — Context diagram

Single system boundary: DRAE exchanges data with residents, CDRRMO administrators, and optional external information services. Central persistence is modeled inside the platform at higher levels. 

flowchart LR
  subgraph ext["External entities"]
    E1["Resident / household user"]
    E2["CDRRMO admin & field staff"]
    E3["External info sources\n(weather APIs, static guidance)"]
  end

  S(("DRAE system\n(Mobile + Admin + API)"))

  E1 <-->|"Credentials, profile,\nincident reports,\nadvisories, hotlines"| S
  E2 <-->|"Residents, staff, reports,\nadvisories, hotlines,\nev centers"| S
  E3 -->|"Supplemental weather /\nreference payloads"| S

Level 0 hides internal processes and data stores; it answers what interacts with the system, not how work is split inside it.
