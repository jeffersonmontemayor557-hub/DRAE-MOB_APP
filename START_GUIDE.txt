DRAE MOBILE APP + ADMIN WEB START GUIDE

1. MOBILE APP

Project folder:
C:\xampp\htdocs\DRAE MobApp\drae-mobapp-app

Use this if new changes do not appear in Expo Go:
cd "C:\xampp\htdocs\DRAE MobApp\drae-mobapp-app"
npx expo start -c

Normal start:
cd "C:\xampp\htdocs\DRAE MobApp\drae-mobapp-app"
npx expo start

Then:
- open Expo Go on your Android phone
- scan the QR code

Important:
- if the profile avatar picker is not visible yet, stop Expo first
- start again with: npx expo start -c
- this clears the old Metro cache and loads the latest screen changes
- advisory push notifications are limited in Expo Go (SDK 53+)
- to test advisory notifications fully, use a development build instead of Expo Go

2. ADMIN WEB

Project folder:
C:\xampp\htdocs\DRAE MobApp\drae-admin-web

Start dev server:
cd "C:\xampp\htdocs\DRAE MobApp\drae-admin-web"
npm run dev

Then open the local URL shown in terminal.
Usually Vite shows something like:
http://localhost:5173

Use .env with the same VITE_SUPABASE_URL / VITE_SUPABASE_ANON_KEY as your Supabase project.
To create mobile app logins without the Supabase Dashboard: open “App logins” in the sidebar, pick a resident, enter a login email, submit. The site shows a one-time temporary password—copy it for the resident; they sign in once, then the app prompts them to set a new password. Apply the Supabase migration that adds profiles.must_change_password if you have not already. Disable “Confirm email” under Authentication → Providers → Email while testing.

3. IF COMMANDS ARE NOT RECOGNIZED

If node or npm is not recognized, open a new terminal window first.
If needed, use:
$env:Path = "C:\Program Files\nodejs;" + $env:Path

Then run the commands again.

4. SUPABASE LOGIN (INVALID CREDENTIALS)

If the app says invalid login credentials but the password looks right:
- SQL seed files do not create Auth users. Either use admin web → “App logins”, or add users in Supabase Dashboard → Authentication → Users (with “Auto Confirm User” or email confirmation disabled for dev).
- If you used the Dashboard only, run drae-mobapp-app/supabase/link_demo_auth_users.sql to attach users to seeded profiles. See DEMO_LOGIN_CREDENTIALS.txt.

5. SUPABASE: APPLY SQL MIGRATIONS (UPLOAD / API ERRORS)

If the mobile app shows errors like:
- column ... does not exist
- Could not find the 'latitude' column of 'incident_reports' in the schema cache (PGRST204)

the hosted database is behind the repo. Open Supabase Dashboard -> SQL Editor and run the
files under drae-mobapp-app/supabase/migrations/ that you have not applied yet (in date order),
or paste the snippets below.

incident_reports GPS (required for incident submit + offline queue sync):
  alter table public.incident_reports
    add column if not exists latitude double precision;
  alter table public.incident_reports
    add column if not exists longitude double precision;

advisories verified flag (required for Weather / advisories fetch):
  alter table public.advisories
    add column if not exists is_verified boolean not null default true;

Confirm you are in the same Supabase project as drae-mobapp-app/.env (EXPO_PUBLIC_SUPABASE_URL).
After SQL runs: Table Editor -> incident_reports -> you should see latitude and longitude columns.
Wait a few seconds for the API schema cache, then tap Sync again on My Reports (or retry submit).
